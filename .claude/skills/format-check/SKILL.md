Zenn記事のフォーマットと日本語品質をチェックする。

## 対象ファイルの特定

引数 `$ARGUMENTS` が指定されている場合はそのパスを使う。
指定がない場合は `git status` で変更中の `.md` ファイルを自動検出する。

## チェック手順

対象ファイルを Read ツールで読み込み、以下を順にチェックする。

### 1. Frontmatter チェック

- `title`, `emoji`, `type`, `topics`, `published` がすべて存在するか
- `type` は `"tech"` または `"idea"` か
- `topics` は配列で1〜5個か
- `published` は boolean か

### 2. Markdown 構文チェック

- コードブロック ` ``` ` に言語名が指定されているか（例: ` ```java `, ` ```bash `）
- 見出しが `#` → `##` → `###` の順で飛び級していないか
- リストのスタイルが混在していないか（`-` と `*` の混在など）
- Zenn固有の `:::message` や `:::details` ブロックが正しく閉じられているか

### 3. 日本語文体チェック

- です・ます調と体言止め・だ・である調が同一記事内で混在していないか
- 句読点は「、。」で統一されているか（「，．」が混入していないか）
- 二重否定（「〜でないことはない」など）が使われていないか
- 冗長表現（「〜することができます」→「〜できます」など）がないか
- 同じ語句の繰り返しが不自然でないか

### 4. 記事構成チェック

- 導入（何を書くか）→ 本文 → まとめ（または結論）の流れがあるか
- 見出しの順序が論理的か
- 長い節（3000字以上）が見出しなしで続いていないか

## 報告形式

問題が見つかった場合は以下の形式で報告する。

```
## フォーマットチェック結果

### 問題あり
- [カテゴリ] 問題の説明（該当箇所を引用）
  → 修正案: ...

### 問題なし
- ...（問題のなかった項目）
```

問題がない場合は「問題は見つかりませんでした」と報告する。
すべて日本語で出力する。
